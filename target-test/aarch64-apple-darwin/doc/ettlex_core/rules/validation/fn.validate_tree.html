pub fn validate_tree(store: &Store) -> Result<()>
Expand description

Validate the entire refinement tree

Runs all invariant checks and returns an error if any violations are found. This is a comprehensive validation that checks:

  • No cycles in parent chains
  • All parent_ids point to existing Ettles
  • All children have corresponding EP mappings
  • No duplicate EP ordinals within an Ettle
  • No children are mapped by multiple EPs (one-to-one constraint)
  • All EP child_ettle_ids point to existing Ettles

Deleted Ettles and EPs are ignored during validation.
